The report describes the results of an accessibility evaluation conducted on www.FabFitFun.com. Also, provided in this report are recommendations for bringing FabFitFun website into compliance with the World Wide Web Consortium’s Web Accessibility Initiative (W3C WAI) for Web Content Authoring Guidelines (WCAG), version 2.1. The WCAG can be found at this URL: https://www.w3.org/TR/WCAG21/. Below is a summary of the project goals and scope.

Project Goals

The goals of this project are to conduct an accessibility evaluation of www.FabFitFun.com using three evaluation methods:

Automated code inspection: using SortSite, a PowerMapper proprietary online tool, we will check the website for broken links, accessibility issues, web standards along with spelling errors and browser compatibility.

Manual code inspection: we will perform a manual code inspection to address areas of coding that might not be covered by the automated tool, such as the accuracy of ALT text and the overall layout of the page;

Executive Summary

Overview

This report describes the results of an accessibility evaluation conducted on www.FabFitFun.com along with our findings and recommendations for bringing the website into compliance with WCAG, version 2.1.

Timeframe, Scope and Methodology

From 04/16/20 to 04/28/20 the accessibility evaluation was performed by using 3 different approaches: automatic code inspection, manual code inspection, and walkthrough experience. The pages evaluated are:

  • Main page
  • Product search under shop page
  • Product detail page

Results: Automated & Manual Code Inspection

54 of 78 existing checkpoints were applicable to the site, and 38 of those 54 points passed on the inspection.

Common failures

  • Keyboard accessibility
  • Missing Non-text content (Alt ” “)
  • Structural gaps (relationship among assets, focus order, parsing and UI components, duplicated IDs, incomplete button labeling)

Recommendations

  • Make the site entirely keyboard accessible, regardless whether the user has the VoiceOver feature active or not. Not every person with grasping issues, needs VoiceOver assistance.
  • Correct all duplicated IDs;
  • Improve the color ratio so users with visual disabilities can navigate the site properly;
  • Add a detailed description of all images so screen readers can describe them to those visually impaired;
  • Identify buttons properly to prevent screen readers from reading the same word twice;
  • Identify the number next to the add to cart button as quantity so screen readers can properly describe the content to the user;
  • Identify from and to on sale items so the screen reader can properly read the strikethrough fonts;
